# Módulo 01 - Exercícios: Fundamentos da Agilidade

## Exercício Prático 1: Analisando Cenários

**Objetivo:** Identificar se uma situação reflete um pensamento Ágil ou Tradicional.

**Instruções:**
Para cada cenário abaixo, classifique como "Ágil" ou "Tradicional" e justifique sua resposta com base nos 4 Valores do Manifesto. Crie um arquivo de texto simples ou Markdown para suas respostas.

1. **Cenário A:** O cliente pediu uma alteração importante no fluxo de pagamento. O Gerente de Projeto respondeu: "Sinto muito, mas os requisitos foram congelados há 3 meses. Só podemos ver isso na próxima fase de contrato, ano que vem."
2. **Cenário B:** A equipe decidiu lançar uma versão simplificada do cadastro hoje para ver se os usuários conseguem usar, em vez de esperar mais 2 semanas para lançar a versão com todas as validações complexas.
3. **Cenário C:** A equipe de desenvolvimento e os especialistas de negócio sentam juntos todos os dias para tirar dúvidas rápidas e ajustar o produto em tempo real.

<details>
<summary>Ver Gabarito Sugerido</summary>

1. **Tradicional.** Viola "Responder a mudanças mais que seguir um plano". O foco está no contrato e no congelamento de escopo, não no valor para o cliente.
2. **Ágil.** Alinha-se com "Software em funcionamento mais que documentação abrangente" e "Responder a mudanças". Prioriza entrega de valor cedo para obter feedback.
3. **Ágil.** Exemplo clássico de "Colaboração com o cliente" e "Indivíduos e interações".
</details>

## Exercício Prático 2: Reescrevendo Contratos Mentais

**Objetivo:** Praticar a mudança de mindset.

**Instruções:**
Imagine que você recebeu as seguintes frases de um colega com mindset fixo/tradicional. Reescreva-as para refletir um mindset ágil.

1. "Não podemos começar a codificar até que tenhamos a especificação de requisitos 100% detalhada e aprovada para evitar retrabalho."
2. "O projeto foi um fracasso. Entregamos o software funcionando, mas atrasamos 2 dias em relação ao cronograma original de 6 meses."
3. "Não fale com o cliente diretamente, isso bagunça o processo. Tudo deve passar pelo gerente."

<details>
<summary>Ver Gabarito Sugerido</summary>

1. *Sugestão:* "Vamos começar implementando o que já entendemos bem e validar com o cliente. O retrabalho estratégico baseado em feedback é melhor do que construir a coisa errada baseada em suposições."
2. *Sugestão:* "O projeto foi um sucesso pois entregamos software funcional que resolve o problema do cliente. O pequeno desvio de prazo é aceitável dado o valor entregue."
3. *Sugestão:* "A comunicação direta com o cliente é a maneira mais eficiente de entender o que precisa ser feito. Vamos facilitar essa interação para evitar telefone sem fio."
</details>
