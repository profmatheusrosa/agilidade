# Módulo 02 - Projeto: Simulando uma Sprint

## Descrição do Projeto
Você irá simular o planejamento e a execução (teórica) de uma Sprint para criar um produto simples: **Um sanduíche perfeito**.

## Cenário
Você é o "Time Scrum" de uma lanchonete ágil.
*   **Product Owner:** O Cliente faminto (você mesmo interpretando o papel).
*   **Developers:** Os cozinheiros (você também).
*   **Scrum Master:** O garantidor do processo (você também).

## Etapas do Projeto

### 1. Product Backlog (O Menu de Desejos)
Liste 5 a 10 itens que poderiam compor sanduíches. Ordene-os por "valor" (gostosura/fome). Dê estimativas de complexidade (P, M, G) para preparar cada item.
*   Ex: Pão Artesanal (M), Queijo (P), Hambúrguer Caseiro (G), Alface (P), Molho Especial (M).

### 2. Sprint Planning
Defina o "Objetivo da Sprint": **"Matar a fome com um clássico saboroso em 10 minutos".**
Selecione itens do Backlog que cabem nesse tempo (Sprint Backlog).
*   *Pergunta:* Você consegue fazer o Hambúrguer Caseiro (G) em 10 minutos junto com o pão? Se não, negocie. Talvez o objetivo mude para "Um lanche rápido frio".

### 3. Execução (A Sprint)
Imagine a execução. Ocorreu algum impedimento? (Acabou o gás? O pão estava velho?).
Como Scrum Master, como você removeria esse impedimento?

### 4. Sprint Review
O sanduíche está pronto. O queijo derreteu? O cliente (você) está feliz? O que faltou?
*   *Resultado:* O cliente achou seco. Novo item no Backlog: "Mais molho".

### 5. Sprint Retrospective
Como foi o processo de fazer o sanduíche?
*   *O que foi bom:* Todos os ingredientes estavam à mão.
*   *O que pode melhorar:* Sujei muita louça.
*   *Plano de Ação:* Usar menos vasilhas na próxima vez.

## Entregável
Escreva um breve relato (diário de bordo) dessa simulação. O importante é vivenciar a estrutura dos eventos e artefatos, mesmo que em um contexto lúdico.
